# Security Policy

## Public boundary

This repository is a sanitized public showcase.

It must not contain:

- private prompts;
- real operational checkpoints;
- raw conversation history;
- personal data;
- sensitive project history;
- private activation packages;
- internal governance mechanisms in full;
- credentials, tokens, keys, or secrets.

## Responsible disclosure

If you find sensitive material accidentally exposed in this repository, open a private contact channel with the maintainer instead of creating a public issue that repeats the sensitive content.

## Maintainer rule

Before publishing any new material, verify:

```text
1. Is this public?
2. Is this sanitized?
3. Does it expose private operational logic?
4. Does it include personal data?
5. Does it copy the private core?
6. Is it useful as public documentation?
```

If the answer is unsafe, do not publish.
